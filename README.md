# FEQUA — Data-Local, Fair, Uncertainty-Aware Breast Cancer Screening

A reproducible research pipeline that learns an interpretable **surrogate of a
sequential (POMDP) breast-cancer screening policy** across heterogeneous
institutions, controls subgroup error, and allocates clinician review through
calibrated conformal deferral. All data are **synthetic** and generated by the
included simulator with fixed seeds, so the whole study reproduces from scratch.

> This repository was rebuilt during revision. See **`CORRECTIONS_NOTE.md`** for a
> point-by-point map from each reviewer/blueprint comment to the code and result
> that addresses it, and **`MODEL_CARD.md`** / **`DATA_CARD.md`** for scope.

## What the pipeline evaluates
- **Federation, done two ways.** A data-local **federated ensemble** (honest
  soft-vote of local LightGBM models) *and* a genuine **FedAvg / FedProx**
  parametric model (true weight averaging + proximal term), against centralized
  (upper reference) and local-only (lower reference) baselines.
- **Decision value, not just accuracy.** Oracle **regret**, capacity/modality/
  budget outcomes, and status-quo comparators (annual, biennial, risk-stratified).
- **Cancer benefit–harm.** Early detection, interval cancers, stage at detection,
  false positives, and benign biopsies via a **closed-loop policy rollout** on
  identical patient primitives.
- **Fairness.** Per-action **TPR and FPR** gaps, oracle-by-group audit, and
  group-aware vs group-blind mitigation across clean / induced-bias / natural
  scenarios.
- **Uncertainty.** Split conformal with class-conditional and Mondrian/subgroup
  coverage, Wilson CIs, set-size/deferral workload, and distribution-shift tests.
- **Robustness.** Site-count (K = 2/5/10/20), non-IID (Dirichlet), and site-size
  imbalance sweeps with divergence measured explicitly.
- **Statistics.** Repeated **patient-level** splits with means, SDs, 95% CIs, and
  paired effect-size tests throughout.

## Quick start
```bash
pip install -r requirements.txt
bash run.sh            # simulate -> experiments -> figures -> tests
```
Or step by step:
```bash
python src/data_simulation.py   # -> data/*.csv, data/*_patients.csv
python src/experiments.py       # -> results/*.json
python src/make_figures.py      # -> figures/*.png
python tests/test_pipeline.py   # unit tests (8/8)
```
Run a single analysis: `python src/experiments.py fairness conformal`.

```

├── requirements.txt        pinned environment

├── CORRECTIONS_NOTE.md     comment-by-comment revision response
├── src/

│   ├── federated.py        federated ensemble + genuine FedAvg/FedProx
│   ├── fairness.py         TPR/FPR gaps, oracle audit, group-aware vs blind
│   ├── conformal.py        split CP, Mondrian, class-conditional, shift
│   ├── decision_analysis.py regret, capacity, benefit-harm
│   ├── rollout.py          closed-loop counterfactual policy rollout

├── tests/test_pipeline.py  splitting / group mapping / conformal quantile tests
├── data/                   synthetic cohorts (regenerated)

└── figures/                all figures (regenerated)
```

## Result files
`classification`, `regression`, `calibration`, `leakage_ablation`,
`heterogeneity`, `fairness`, `conformal`, `decision`, `benefit_harm`,
`incremental`, `shap` (each in `results/`). Figures `fig1`–`fig10` in `figures/`.

## Scope
Simulation-based methods research. It does **not** establish clinical
effectiveness, real-world cancer-outcome improvement, or formal privacy; data
locality is not a privacy guarantee. See the cards for details.
